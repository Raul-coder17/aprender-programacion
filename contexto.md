1. Sintaxis y SemánticaAntes de escribir tu primera línea, debes conocer esta diferencia:Sintaxis: Son las reglas gramaticales del lenguaje; dictan cómo debes escribir las instrucciones para que la computadora las reconozca como válidas.  Semántica: Es el significado real de esas instrucciones o lo que hacen al ejecutarse.  Consejo: Entender la diferencia entre ambas es crucial a la hora de buscar y arreglar errores (debuggear). Escribir let edad = 20; es usar la sintaxis correcta para darle la instrucción semántica a la máquina de "guardar el número 20 en la memoria".  2. Variables y Tipos de DatosLos datos con los que vas a trabajar se agrupan en distintas categorías.Tipos Primitivos:Números enteros (valores sin decimales) y números decimales (valores con precisión limitada).  Booleanos: Pueden ser verdadero o falso, siendo la base de toda lógica.  Caracteres (símbolos individuales de texto) y valores Nulo/Vacío (que representan una ausencia intencional de valor).  Tipos Compuestos:Cadenas de texto (secuencias de caracteres) y Arrays o Listas (colecciones ordenadas de elementos).  Objetos (agrupaciones de datos relacionados) y Tuplas (colecciones inmutables que tienen un tamaño fijo).  Sistemas de Tipos: Dependiendo del lenguaje que uses en tu editor, el tipado puede ser estático (los tipos se verifican en tiempo de compilación) o dinámico (se verifican durante la ejecución). También existe el tipado fuerte (las conversiones entre tipos deben ser explícitas) o el tipado débil (las conversiones ocurren de forma automática).  3. OperadoresSon los símbolos que permiten procesar y evaluar los datos.Aritméticos: Suma (+), resta (-), multiplicación (*), división (/), módulo (%), y potencia ().  De comparación: Igualdad, desigualdad, mayor que, menor que (==, !=, <, >, <=, >=).  Lógicos y de asignación: Evalúan condiciones con AND, OR, NOT, o asignan valores combinados como =, +=, -=. También existe el operador ternario, que evalúa una condición ? y devuelve un valor si es verdadera o falsa :.  4. Estructuras de ControlEstas estructuras dictan qué camino toma el flujo de tu programa.Condicionales: Permiten ejecutar código diferente dependiendo de ciertas condiciones. Para esto usarás if/else (el caso más básico) o switch/match (muy útil cuando tienes múltiples casos a evaluar).  Bucles:for: Úsalo cuando ya conoces el número exacto de iteraciones.  while: Empléalo cuando la condición para detener el bucle sea dinámica.  for-each / for-in: Son especiales para iterar sobre colecciones como los Arrays.  do-while: Úsalo cuando necesites garantizar que tu bloque de código se ejecute al menos una vez.  Control de Flujo Interno: Comandos como break (para salir de un bucle completamente), continue (para saltarse la iteración actual y pasar a la siguiente) y return (para terminar una función devolviendo un valor).  5. FuncionesSon el mecanismo principal en programación para la abstracción y la reutilización de código.  Reglas de oro: Una buena función hace una sola cosa, tiene un nombre puramente descriptivo y es fácil de testear de forma aislada.  Conceptos que debes dominar: Aprende a configurar la declaración (nombre, parámetros y cuerpo), trabajar con parámetros (por valor, por referencia o valores por defecto) y dominar el "scope" o ámbito de las variables. Además, la guía destaca las funciones anónimas y la recursividad (cuando las funciones se llaman a sí mismas).  💻 Ejercicios Prácticos para tu EditorPuedes resolver estos ejercicios utilizando el lenguaje que prefieras (JavaScript, Python, C#, etc.):Ejercicio 1: Variables y OperadoresCrea una variable con un tipo primitivo numérico entero y otra con un número decimal.
